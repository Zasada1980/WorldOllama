# Правило хранения документации

Все новые документы (архитектура, ADR, справка, гайды, схемы) должны размещаться строго в каталоге `AGENTS/docs`. Не создавайте документационные `.md` файлы вне этой директории.

## Цель
Единая точка правды, упрощение навигации и предотвращение рассредоточения знаний.

## Область действия
- Markdown (`.md`)
- Текстовые описания (`.txt`)
- Диаграммы (`.drawio`, `.png`, `.svg`) относящиеся к архитектуре
- ADR файлы

## Исключения
- Встроенные README для внешних зависимостей или vendored-кода — допускаются, но должны ссылаться на основной раздел в `AGENTS/docs`.

## Конвенции
- Категории: `architecture/`, `guides/`, `reference/`, `decisions/`, `changelog/`.
- Имена: kebab-case, без пробелов.
- ADR: `YYYY-MM-DD-topic.md`.

## Проверка
При добавлении нового файла вне `AGENTS/docs` с расширением `.md` — переместить в `AGENTS/docs`.

## Обновление правил
Изменения в этой инструкции вносятся через PR / отдельный коммит с описанием причины.
