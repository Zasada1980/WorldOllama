# Документация проекта AGENTS

Эта директория (`AGENTS/docs`) является единственным местом для всей проектной документации.

## Структура
- `README.md` — общий обзор.
- `architecture/` — схемы, описания компонентов.
- `guides/` — пошаговые инструкции (установка, запуск, расширение).
- `reference/` — справочные материалы (API, форматы данных, протоколы взаимодействия).
- `decisions/` — ADR (Architecture Decision Records).
- `changelog/` — история изменений (если не используете git tags для этого).

## Правила
1. Новые документы добавляются только сюда, не в другие каталоги.
2. Используйте понятные имена файлов: `api-overview.md`, `agent-lifecycle.md`.
3. Для архитектурных решений добавляйте префикс даты: `2025-11-21-choice-of-vector-store.md`.
4. Поддерживайте короткий формат: цель → контекст → решение → последствия.
5. Обновляйте/архивируйте устаревшие документы, не удаляя важную историю решений.

## Добавление нового документа (рекомендуемый формат)
```
# Заголовок
Краткое резюме (1–2 предложения)

## Контекст
Почему возникла потребность.

## Варианты
Список рассмотренных альтернатив.

## Решение
Выбранный путь и причины.

## Последствия
Плюсы, минусы, что мониторить.
```

## ADR Пример
```
# 2025-11-21 Использование SQLite для локального состояния
Контекст: требуется лёгкое локальное хранилище.
Решение: выбрать SQLite (файл `OllamaModels/db.sqlite`).
Причины: простота, отсутствие доп. установки.
Последствия: ограничение по параллельной записи; возможный переход на Postgres позже.
```
